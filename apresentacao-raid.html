Lucas MendesIntrodução

O que é RAID?

Tecnologia que combina múltiplos discos físicos em uma única unidade lógica — melhorando desempenho, confiabilidade ou ambos.

  • Criado em 1987 por Patterson, Gibson e Katz na UC Berkeley
  • Implementado via hardware (controladora RAID) ou software (Sistema Operacional)
  • Diferentes níveis oferecem diferentes balanços entre performance, capacidade e redundância
  • RAID 10 lidera deployments enterprise com 35% — seguido pelo RAID 6 com 22%
  • Mercado global de NAS atingiu $34,5 bilhões em 2024
Lucas MendesRAID 0

RAID 0 — Strip Set

  • Divide dados em blocos distribuídos entre dois ou mais discos (striping)
  • Discos operam em paralelo como um único volume lógico
  • Capacidade = soma de todos os discos (2 × 1TB = 2TB)
  • Sem redundância e sem paridade — perda de um disco compromete tudo
  • Mínimo de 2 discos
Onde é utilizado
PCs gamer de alto desempenho Edição de vídeo e áudio profissional Renderização 3D e workstations criativas
Lucas MendesStriping

Striping — Divisão de Dados

  • Dados divididos em blocos (stripes) e distribuídos entre múltiplos discos
  • Strip Set (sem paridade): RAID 0 — foco total em performance
  • Strip Set c/ paridade: RAID 5 — performance + proteção de dados
  • Leituras e escritas ocorrem em paralelo → velocidade multiplica pelo nº de discos
  • Quanto mais discos, maior a performance — e maior o risco sem paridade

"Como dividir a entrega de um pedido entre 2 motoboys ao mesmo tempo — mais rápido, mas se um sumir, o pedido chega incompleto."

Lucas MendesMirroring

Mirroring — Espelhamento

  • Dados gravados identicamente em dois ou mais discos em tempo real
  • Cada disco é uma cópia exata do outro — espelho perfeito
  • Se um disco falhar, o outro continua operando sem interrupção
  • Capacidade utilizável = tamanho de um único disco
  • Base do RAID 1 — foco em confiabilidade e disponibilidade

"Como tirar xerox de documentos importantes e guardar em dois lugares diferentes — se um sumir, o outro está intacto."

Redundância & Paridade

Redundância
Garantia de disponibilidade
  • Garante funcionamento contínuo dos serviços
  • Evita perda de dados em caso de falha de hardware
  • Fontes de energia alternativas
  • Múltiplos locais de armazenamento de dados
Paridade
Detecção e correção de erros
  • Detecta erros nas transmissões de dados
  • Um bit extra é anexado a cada byte transmitido
  • Se a paridade não coincidir, um erro é detectado
  • Permite reconstruir dados de discos com falha
Exemplos do cotidiano
Nobreak / UPS — mantém servidores ligados durante queda de energia Backup em nuvem — Google Drive, iCloud, OneDrive Gerador de emergência — hospitais e data centers Internet redundante — fibra + 4G simultâneos Fonte de alimentação dupla em servidores

Lucas MendesRAID 1

RAID 1 — Mirror

  • Dados gravados identicamente em dois ou mais discos (espelhamento)
  • Capacidade = tamanho de um único disco (2 × 1TB = 1TB utilizável)
  • Requer discos com a mesma capacidade de armazenamento
  • Alta tolerância a falhas — se um disco falhar, o outro continua operando
  • Mínimo de 2 discos
Onde é utilizado
Servidores de banco de dados (MySQL, PostgreSQL) Sistemas hospitalares — prontuários eletrônicos Sistemas financeiros e bancários críticos

RAID 2 · RAID 3 · RAID 4

João AugustoRAID 2

RAID 2 — Bit-level striping com ECC

  • Striping a nível de bit — dados distribuídos bit a bit entre discos
  • ECC (Código de Correção de Erros) no lugar de paridade simples
  • Discos dedicados para armazenar código Hamming ECC
  • Raro em produção — complexidade e custo muito elevados
  • Substituído pelo RAID 3, 4 e 5 em ambientes modernos
Onde é utilizado
Sistemas científicos com correção crítica de erros Aplicações de nicho especializadas
João AugustoRAID 3

RAID 3

  • Striping a nível de byte com disco dedicado exclusivamente para paridade
  • Um bit de paridade é computado para cada palavra de dados
  • Paridade armazenada em um disco separado dos dados
  • Alta performance em leituras sequenciais de arquivos grandes
  • Gargalo no disco de paridade durante escritas
  • Mínimo de 3 discos (2 dados + 1 paridade)
Onde é utilizado
Edição de vídeo HD sequencial Armazenamento científico de dados Streaming de grandes arquivos
João AugustoRAID 4

RAID 4

  • Similar ao RAID 3, mas paridade opera em blocos maiores (block-level striping)
  • Um disco exclusivo armazena todas as informações de paridade
  • Em falha de disco, dados reconstruídos pela paridade dos demais
  • Disco de paridade se torna gargalo nas operações de escrita
  • Mínimo de 3 discos
Onde é utilizado
NAS (Network Attached Storage) Servidores de leitura intensiva

RAID 5 · RAID 6

Jhonatan ZagoRAID 5

RAID 5 — Strip Set c/ Paridade

  • Paridade distribuída entre todos os discos — sem disco dedicado
  • Capacidade = (n − 1) discos; 1 disco equivalente reservado para paridade
  • Tolera falha de 1 disco — dados reconstruídos pela paridade dos demais
  • Melhor balanço entre performance, capacidade e redundância
  • Mínimo de 3 discos
Onde é utilizado
NAS corporativo (Synology, QNAP) Servidores de arquivos empresariais Servidores web e storages empresariais
Jhonatan ZagoRAID 6

RAID 6 — Dupla Paridade

  • Dois blocos de paridade independentes (P e Q) distribuídos entre os discos
  • Suporta falha simultânea de até 2 discos sem perda de dados
  • P = XOR (como RAID 5); Q = algoritmo de Galois (Reed-Solomon)
  • Capacidade = (n − 2) discos úteis; mínimo de 4 discos
Onde é utilizado
Data centers com arrays de muitos discos Sistemas críticos — tolerância a 2 falhas Backup corporativo de alta disponibilidade

Rhaidy VieiraRAIDs Combinados

RAIDs Associados

É possível associar dois ou mais tipos de RAID para obter o melhor de cada abordagem simultaneamente.

  • RAID 0 + RAID 1 = RAID 10 — desempenho do striping com a segurança do espelhamento
  • RAID 5 + RAID 0 = RAID 50 — paridade distribuída com extensão de volume
  • A combinação permite maior tolerância a falhas e maior desempenho ao mesmo tempo
  • Exige mais discos físicos para implementação
Rhaidy VieiraRAID 10

RAID 10 — RAID 0 + RAID 1

  • 1ª etapa: RAID 0 — 2 conjuntos de 2 discos de 1TB, criando 2 volumes de 2TB
  • 2ª etapa: RAID 1 — os dois volumes são espelhados, resultando em 2TB utilizáveis
  • Tolerância: 1 disco por grupo — pode perder até 2 discos, 1 em cada grupo
  • Combina máxima performance com alta disponibilidade
  • Mínimo de 4 discos
Onde é utilizado
Bancos de dados de alta performance (OLTP) Servidores de virtualização
Rhaidy VieiraRAID 50 — 1ª Forma

RAID 50 — RAID 5 + RAID 0

1ª Etapa — RAID 5
Strip Set c/ paridade
  • 2 conjuntos de 3 discos (1TB cada)
  • Paridade distribuída em cada conjunto
  • Volume de 2TB em cada conjunto (1TB para paridade — 1/3)
2ª Etapa — RAID 0
Strip Set
  • Os dois volumes de 2TB são somados por striping
  • Volume final: 4TB utilizáveis
  • Tolerância: 1 disco por conjunto — até 2 discos no total
Rhaidy VieiraRAID 50 — 2ª Forma

RAID 50 — RAID 0 + RAID 5

1ª Etapa — RAID 0
Strip Set
  • 3 conjuntos de 2 discos (1TB cada)
  • Extensão de volume em cada conjunto
  • Volume de 2TB por conjunto
2ª Etapa — RAID 5
Strip Set c/ paridade
  • Paridade distribuída entre os 3 volumes de 2TB
  • Volume final: 4TB utilizáveis
  • Menos viável — perda de 1 disco em qualquer conjunto compromete o volume RAID 0 daquele grupo
